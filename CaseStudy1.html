Case Study 1 - Manuel Diaz Bendito -  Stout:


Dataset description:

The provided dataset contains the Loan data from a Lending Club. It represents thousands of loans made through the Lending Club platform, which allows individuals to lend to other individuals. It contains 10000 instances (or individuals), and 55 features (or variables) related to those individuals. 

Some of the features the dataset contains are the job title of the individual (emp_title), the number of years in the job (emp_length), among others. This shows that there are numerical features, as the "emp_length", and text features, as the "emp_title". Through feature engineering, this last kind of features could be turned into numerical feature to be used in a training process. However, for this Case Study, they have been removed to simplify the data processing.

On the other hand, there are many instances in which the numerical features are assigned as not a number (nan). In these cases, in order not to ruin the whole instance, those "nan" occurrences have been turned into extreme outliers. Thus, a value of -10000 has been assigned to them.

Visualizations:

The visualization process is a very important procedure in Artificial Intelligence, as it shows the relationship of data and lets the data scientist decide which model best fits the dataset being analyzed. For that reason, a good idea is to plot each feature with all the others and see their visual relation. In this case, as we were asked to predict the feature named "interest_rate", every other feature was plotted with the target one to see their relation.

This plotting process was very interesting, as it was performed before the cleansing process in which non numeric features were removed from the dataset. To do that, a try-except syntax was used to avoid errors.

These interesting relations gave the idea to train one specific Machine Learning model: a k-Nearest Neighbors using the clustering feature.

Feature set creation and model training process - kNN model:

As explained above, the data cleansing described was performed. However, there was a non numerical feature that, the "grade" feature, that showed a clear clustering relationship with the target feature, and feature engineering was applied to it. This feature engineering assigned a number to each one of the grade letters, in order, i.e. 'A' = 1, 'B' = 2, and so on.

Thus, this kNN model was trained using only that feature, to show the prediction possibilities of just one feature. This was made not only because of the reduction of the model complexity, but also because as a kNN model uses all the dataset for every prediction, using a large number of features would have increased the amount of memory the model would require to work.

With all the above, the dataset was split in a train set (70%), a test set (15%) and a validation set (15%). Then, a hyperparameter tuning process was performed, changing the k value from 1 to 100 to reach the best validation score. In the following feature, the evolution of the validation score when changing the k value is shown:

The figure shows not only the outstanding validation scores obtained, but also that increasing the k value increased the validation score drastically at the beginning. 

With all the above, the final test score obtained was 94.6%, as shown in the following figure:

Finally, to enhance this mode, if more time was available, I would have added other features to the "grade" feature in a feature selection process in order to see if any of the not considered features increased the test accuracy of the model. Also, other hyperparameters like the type of distance could be tuned to obtain even better results.

Feature set creation and model training process - Random Forest model:

Once the kNN was trained and the good results obtained, I decided to train a RF with a larger number of features and training and tuning time in order to see if I could get even better results.

To do so, I used all the features that were non numerical (removing also the "grade" feature), and changed every "nan" value to -10000. Also, I performed a grid search process in which, given a set of possible hyperparameters, I trained several MLP models and validated them with a cross-validation procedure of 10 folds to get the best set of hyperparameters possible. The results are the ones shown in the following figure:

In the figure, it is shown the final hyperparameters of the best model obtained. Also, the best train and validation scores are shown (there are 10 values because of the cross-validation process), and the final best test score is shown, which was a score of 90.67%.

Even though the final test score was lower than the kNN model, a deeper grid search would have eventually reached the outstanding kNN model result. This is the main enhancement that I would implement for this model: a deeper grid search with more time to train and validate, which was limited due to the heavy processing time that a model as RF's have.

Conclusion:

The provided dataset had to be approached by visualizing the data because of a very clear clustering relationship between one of the features and the target feature. However, brute force and plenty of computational time will also give outstanding results when predicting the target feature.
